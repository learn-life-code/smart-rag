🔍 RTC Analyzer — Setup Guide

Copy the folder to your PC, follow the steps, and you're ready. No coding needed.

What this is: The RTC Analyzer reads a bug folder and finds the root cause. It works best when it has a search database (.db) built from the software — that lets it answer in seconds with cited sources instead of reading everything slowly. This guide gets you from a fresh PC to running, and shows you exactly what to do if the database doesn't exist yet.

Part 1 · One-time setup (do this once per PC)

1

Copy the whole folder to your PC

Copy the entire Industrialization folder anywhere on your machine (e.g. C:\Tools\Industrialization). Keep all files together.

⚠ If you copied a .venv folder from another PC, it won't work here — the next step rebuilds it correctly.

2

Run the setup script

Open PowerShell in the folder (Shift + right-click → "Open PowerShell here"), then paste:

powershell -ExecutionPolicy Bypass -File .\setup_venv.ps1

Behind a company network/proxy? Add your proxy:

powershell -ExecutionPolicy Bypass -File .\setup_venv.ps1 -Proxy http://rb-proxy-na.bosch.com:8080

It finds the right Python, builds the environment, installs everything, and ends with a clear status. Look for:

  • ✅ READY — you're good
  • Retrieval mode: hybrid = best (semantic search)  ·  keyword-only = still works, just no "how does X work" search

No Python installed? Install Python 3.13 from python.org first, then re-run. The script will tell you if anything is missing.

Part 2 · The database (.db) — the important part

❓ Do you already have a database for your software build?

A database is a file named baseline_distill.db (or a .codegraph/codegraph.db) that sits with your software build.

⬇    pick the branch that matches you    ⬇
A

YES — I have the .db already

Great — just point the RTC Analyzer at it. Launch the app:

run_gui.bat

In the app's Settings, find "CodeGraph DB or Build Root" and paste the path to your build folder (or directly to the .db using the .db button). Then run your bug folder.

✓ Done — the analyzer will use the database automatically.

B

NO — I don't have a .db yet (or not sure)

No problem — create it once from your software folder. It takes about a minute. In PowerShell:

py -3.13 scripts/distill_baseline.py "C:\path\to\your\software\build"

When it finishes it prints a line like:

✅ BASELINE READY
   PASTE THIS PATH into the RTC analyzer:
   ...\.codegraph\baseline_distill.db

Copy that path, launch run_gui.bat, and paste it into "CodeGraph DB or Build Root" (Branch A above). That's it.

What if it says some files need CodeGraph?

If your build has firmware binaries (like .elf/.img/.mbn), you'll see a message: ⚠ Run codegraph first. Those are compiled files the analyzer can't read directly. Run CodeGraph on the folder first, then re-run the command above — it will include them automatically.

Part 3 · How do I KNOW it has a database? (visual check)

Run this any time to see what's set up:

py -3.13 -m smart_rag.cli indexes
What you seeMeaningDo
a name + entities/facts listed Database exists ✓Use it (Branch A)
"No indexes" No database yetCreate it (Branch B)

The databases live in C:\Users\<you>\.smartrag\indexes\ (central) or next to your build folder. The analyzer finds them by the path you set in Settings.

Part 4 · Run a bug analysis

1

Open the analyzer

run_gui.bat
2

Set the database path once (Settings)

Paste your build/​db path into "CodeGraph DB or Build Root".

3

Point it at a bug folder and analyze

The analyzer reads the logs, uses the database for fast cited evidence, and produces the root cause.

The 30-second summary:
  1. Copy folder → run setup_venv.ps1 (once).
  2. Have a .db? Paste its path in Settings. Don't have one? Run distill_baseline.py "your-build" to create it.
  3. Run run_gui.bat, pick a bug folder, analyze.
